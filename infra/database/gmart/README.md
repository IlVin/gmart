# 🐘 PostgreSQL Infrastructure (gmart)

Минималистичная конфигурация **PostgreSQL** в Docker для разработки. 
Работает в **хостовой сети** на порту **5550** без постоянных томов (данные удаляются при `make down`).

---

## 🚀 Быстрый старт


| Команда | Описание |
| :--- | :--- |
| `make build` | Сборка локального Docker-образа `gmart_postgres`. |
| `make up` | **Полный перезапуск:** сборка, удаление старого и запуск нового контейнера (чистая база). |
| `make start` | Запуск уже существующего (остановленного) контейнера с сохранением данных. |
| `make stop` | Остановка контейнера (данные сохраняются внутри контейнера). |
| `make down` | **Удаление:** остановка и полное удаление контейнера. **Все данные теряются.** |
| `make psql` | Интерактивный вход в консоль `psql` (внутри контейнера). |
| `make logs` | Просмотр логов (выход через `Ctrl+C`). |
| `make info` | Проверка статуса контейнера и доступности порта 5550. |

---

## ⚙️ Конфигурация по умолчанию

Контейнер использует переменные окружения для автоматической инициализации:

- **Host:** `localhost` (через `--network host`)
- **Port:** `5550`
- **User:** `gmart`
- **Password:** `gmart_pswd`
- **Database:** `gmart`

---

## 🛠 Особенности реализации

1. **Host Network**: Контейнер использует сетевой стек хоста. Порт `5550` пробрасывается напрямую, минуя Docker Proxy.
2. **Healthcheck**: В образе настроен мониторинг состояния через `pg_isready`. Статус `healthy` появится только после полной готовности БД.
3. **Ephemeral Storage**: Данные хранятся в записываемом слое контейнера. Чтобы данные не пропали после удаления контейнера, используйте `make stop` вместо `make down`.
