## Конфигурация (Module: config)

Модуль управления конфигурацией спроектирован как **иммутабельный объект** с поддержкой нескольких источников данных, строгой типизацией и встроенной валидацией.

### Ключевые особенности

*   **Иммутабельность (Immutable State):** Объект конфигурации защищен от случайных изменений в рантайме. Любая модификация (через методы `Set...`) возвращает новую копию объекта. Это исключает возникновение побочных эффектов и "гонок данных" (data races).
*   **Приоритетность настроек:** Система автоматически разрешает конфликты параметров, следуя иерархии (от низшего к высшему):
    1.  Значения по умолчанию (Hardcoded defaults).
    2.  Флаги командной строки (Command line flags).
    3.  Переменные окружения (Environment variables).
*   **Строгая типизация:**
    *   `SocketAddr`: Гарантирует корректность сетевого адреса `host:port` еще на этапе инициализации.
    *   `ShortBaseURL`: Обеспечивает валидность базового URL, автоматически очищая его от чувствительных данных (User Info).
*   **Тестируемость:** Архитектура `NewConfig` позволяет подменять системные вызовы (env, args), что обеспечивает **100% покрытие Unit-тестами** без влияния на глобальное окружение ОС.

### Использование

Модуль поддерживает удобные цепочки вызовов (chaining) и принцип **Fail Fast** через обертку `Must`:

```go
// Инициализация основного конфига
cfg, err := config.NewConfig(&os.Args, os.LookupEnv)

// Создание модифицированной копии для тестов
testCfg := cfg.SetRepoDrv("InMemory").
               SetListenAddr(config.Must(config.NewSocketAddr("127.0.0.1:0")))
