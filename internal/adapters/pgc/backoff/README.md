# Module: Postgres Backoff (pgc)

Пакет предоставляет механизмы для автоматического повтора (retry) операций с базой данных PostgreSQL при возникновении временных сбоев.

---

## Основные возможности

*   **Exponential Backoff**: Автоматический расчет множителя задержки на основе заданного лимита времени и количества попыток.
*   **Jitter**: Добавление случайного отклонения (±10%) к времени ожидания для предотвращения синхронных запросов от множества клиентов.
*   **Умная фильтрация ошибок**: Система знает, какие ошибки PostgreSQL являются временными (Network, Deadlock, Serialization Failure), а какие фатальными (Syntax Error, Constraint Violation).

---

## Использование

### Инициализация и запуск
Создайте объект бекоффа, указав количество попыток и максимальное время суммарного ожидания.

```go
// 3 попытки, максимальная задержка до 2 секунд
backoff := pgc.NewPgBackoff(3, 2*time.Second)

err := backoff.WithRetry(ctx, func() error {
    return repo.UpdateData(ctx, data)
})
```

---

## Логика повторов (Retriable Errors)

Модуль автоматически разрешает повтор для следующих классов ошибок:
1.  **Сетевые ошибки**: Таймауты, обрывы соединений, ошибки сокетов.
2.  **Класс 08 (Connection Exception)**: Ошибки протокола и связи с БД.
3.  **Класс 40 (Transaction Rollback)**: Дедлоки и ошибки сериализации транзакций.
4.  **Класс 57 (Operator Intervention)**: Остановка или краш сервера базы данных.

**Фатальные ошибки** (нарушение уникальности, синтаксис, отсутствие таблиц) прерывают цикл повторов немедленно.

---

## Технические детали

*   **Минимальная задержка**: 100ms.
*   **Логирование**: Каждая попытка повтора фиксируется в `slog` с указанием текущей задержки и оставшегося количества попыток.
*   **Контекст**: Полная поддержка `context.Context`. Если контекст отменяется во время ожидания, метод возвращает агрегированную ошибку всех попыток и `ctx.Err()`.
