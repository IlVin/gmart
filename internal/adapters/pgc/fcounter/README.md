# Module: Failure Counter (fcounter)

Счетчик ошибок на основе кольцевого буфера. Предназначен для реализации логики **Circuit Breaker**: отслеживания частоты сбоев в операциях с базой данных или внешними сервисами.

---

## Основные принципы

Счетчик считает порог превышенным, если в течение заданного интервала времени (`perDuration`) произошло более `maxFailures` ошибок.

*   **Скользящее окно**: Ошибки, вышедшие за пределы интервала времени, перестают учитываться автоматически.
*   **Zero-allocation (в горячем цикле)**: После инициализации метод `Inc` не создает новых аллокаций, перезаписывая старые структуры в кольцевом буфере.

---

## Использование

### Инициализация
```go
// Не более 5 ошибок за последние 10 секунд
counter := fcounter.NewFailureCounter(5, 10*time.Second)
```

### Регистрация событий
При каждой ошибке вызывайте `Inc`. Метод вернет `true`, если лимит превышен.

```go
if err := db.Query(); err != nil {
    if counter.Inc(err) {
        log.Println("Слишком много ошибок! Операции приостановлены.")
    }
} else {
    counter.Reset() // Сбрасываем при успехе
}
```

---

## Методы

*   **Inc(err)**: Регистрирует новую ошибку. Возвращает статус превышения лимита.
*   **IsLimitExceeded()**: Потокобезопасная проверка текущего состояния счетчика.
*   **Error()**: Собирает все актуальные ошибки в окне в одну через `errors.Join`.
*   **Reset()**: Помечает счетчик как "промытый", очищая актуальные записи.
*   **Clear()**: Полная реинициализация буфера.

---

## Технические детали

*   **Минимальный интервал**: 100ms.
*   **Минимальное кол-во ошибок**: 3.
*   **Память**: Используется фиксированный слайс структур `failure`, что минимизирует нагрузку на GC при интенсивной работе.
